# Cookie HTTP Adapter

This document describes the experimental faster adapter in the `feature/cookie-http-adapter` branch.

## Purpose

The default gateway uses the Playwright Web Adapter:

```text
XJGPT frontend -> FastAPI gateway -> Playwright -> XipuAI web page -> answer
```

The cookie adapter is designed as a faster experimental path:

```text
XJGPT frontend -> FastAPI gateway -> httpx -> authorized XipuAI backend endpoint -> answer
```

The adapter reads the local `school_gpt_state.json` file generated by `xjgpt-login`. It does not upload cookie values, print cookie values, or commit login state to GitHub.

## Safety Boundary

Use this adapter only with your own authorized school account and school-approved endpoints.

Do not:

- upload `school_gpt_state.json`;
- paste cookie values into source code;
- use another student's cookie;
- bypass SSO, CAPTCHA, or rate limits;
- use this adapter against services where direct HTTP access is not permitted.

## Required Local Configuration

The cookie adapter needs the real backend endpoint used by XipuAI. This branch does not hard-code that endpoint because it must be confirmed from the authorized school environment.

You can now use the local discovery helper:

```bash
xjgpt-login
xjgpt-discover-endpoint
```

If auto-submit does not work, use manual mode:

```bash
xjgpt-discover-endpoint --manual --timeout 60
```

The helper writes a local report and a suggested env file:

```text
local_discovery/xipuai_endpoint_candidates.json
.env.cookie.local
```

Detailed instructions are in `docs/ENDPOINT_DISCOVERY.md`.

After reviewing the suggested endpoint, set these environment variables locally:

```bash
export XJGPT_ADAPTER_MODE=cookie
export XIPUAI_CHAT_ENDPOINT="/your/authorized/xipuai/chat/endpoint"
```

On Windows PowerShell:

```powershell
$env:XJGPT_ADAPTER_MODE="cookie"
$env:XIPUAI_CHAT_ENDPOINT="/your/authorized/xipuai/chat/endpoint"
```

Optional variables:

```bash
export XIPUAI_BASE_URL="https://xipuai.xjtlu.edu.cn"
export XIPUAI_API_METHOD="POST"
export XIPUAI_PAYLOAD_STYLE="openai"   # openai or simple
export XJGPT_STATE_FILE="school_gpt_state.json"
```

If the backend requires a token stored in localStorage, you can configure:

```bash
export XIPUAI_AUTH_HEADER_FROM_LOCALSTORAGE="token_storage_key_here"
export XIPUAI_AUTH_HEADER_NAME="Authorization"
export XIPUAI_AUTH_HEADER_PREFIX="Bearer"
```

If the backend requires extra headers:

```bash
export XIPUAI_EXTRA_HEADERS_JSON='{"X-Requested-With":"XMLHttpRequest"}'
```

## How to Test

First save your local login state:

```bash
xjgpt-login
```

Check cookies only:

```bash
python scripts/test_cookie_http_adapter.py --check-cookies-only
```

After setting `XIPUAI_CHAT_ENDPOINT`, test an actual call:

```bash
python scripts/test_cookie_http_adapter.py --question "请只回复两个字：成功" --model auto
```

Then start the gateway in cookie mode:

```bash
export XJGPT_ADAPTER_MODE=cookie
export XIPUAI_CHAT_ENDPOINT="/your/authorized/xipuai/chat/endpoint"
xjgpt-gateway --host 127.0.0.1 --port 8000
```

Check active adapter status:

```bash
curl http://127.0.0.1:8000/v1/adapter
```

Send a normal request:

```bash
curl -X POST http://127.0.0.1:8000/v1/chat \
  -H "Authorization: Bearer sk-student-demo-001" \
  -H "Content-Type: application/json" \
  -d '{"model":"auto","messages":[{"role":"user","content":"请只回复两个字：成功"}]}'
```

## Model Behaviour

The gateway still accepts the same model ids:

```text
auto
gpt-5.4
deepseek-r1
deepseek-v3
qwen-max
```

In web mode, the adapter attempts to select the matching model in the XipuAI web page.

In cookie mode, the adapter sends the model id in the HTTP payload. If the real XipuAI backend expects a different model code, set a model override environment variable, for example:

```bash
export XIPUAI_MODEL_GPT_5_4="the-real-backend-model-code"
```

## Current Limitation

The cookie adapter cannot be fully activated until `XIPUAI_CHAT_ENDPOINT` is known. The endpoint discovery helper is designed to identify that endpoint from your local authorized browser session, but you still need to review the suggested result before enabling cookie mode.
